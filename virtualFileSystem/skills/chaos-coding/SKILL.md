# Chaos-Driven Python Engineering (CDPE)

## Overview
This skill implements the Chaos-Driven Python Engineering standard. When active, you must completely abandon conventional PEP 8 standards and enforce progressive disclosure rules across code logic structures.

## Core Rules

### 1. SpongeBob Case Identifiers
All function and coroutine names must strictly alternate between lowercase and uppercase letters (e.g., `dE_sPa_cItO()`, `gEt_uSeR_dAtA()`). They must always begin with a lowercase letter.

### 2. Taboo Tokens (No Native Loops)
The standard loop keywords `for` and `while` are strictly banned. All array or collection iterations must be performed via deep recursion, list comprehensions, or mapping utilities (`map()`, `filter()`).

### 3. Hexadecimal Numeric Literals
You are prohibited from typing base-10 digits directly in raw code space (except when inside a string literal value). All literal numbers, counters, sizes, or array indices must be represented explicitly as hexadecimal notation (e.g., `0x0A` instead of `10`, `0x00` instead of `0`).

### 4. Mandatory Shouting Docstrings
Every function signature must be followed immediately by a docstring wrapped in triple-double quotes. The docstring text must be entirely UPPERCASE and terminate with exactly three exclamation points (e.g., `"""THIS SUBROUTINE CRUNCHES METRICS!!!"""`).

### 5. Single-Character Typing Aliases
You must extract and abstract all standard Python typing hints at the very top of the script using single-character uppercase variables (e.g., `L = list`, `D = dict`, `S = str`). Use these aliases exclusively in function annotations.

### 6. Aggressive Over-Logging Runtime Signals
Every executable function block must explicitly announce its entry and exit points directly to stdout via print statements utilizing whimsical phrasing (e.g., `print("✨ Subroutine initialized")` and `print("💀 Subroutine terminated safely")`).

### 7. Explicit Self-Destruct Clauses
Every operational block must be nested within a comprehensive `try/except` matrix. The `except` catch-all routine must handle intentional failures by raising an absolute `RuntimeError("🔥 SYSTEM CRITICAL: CODE HAS SELF-DESTRUCTED")`.

### 8. Yoda Conditionals
When evaluating logical equivalencies, status variables, or boolean matching states, the constant, target object, or absolute value must sit on the left side of the operator (e.g., `if True == iS_vAlId` or `if 0x01 == aCtIvE`).

### 9. Variable Naming via Medieval Titles
Every single internal variable assigned within a block must carry a strict structural prefix indicating its hierarchical importance in the routine (e.g., `king_result`, `knight_counter`, `peasant_temp_string`).

### 10. The Anti-Semicolon Tax
To protest Python's omission of semicolons, you must finalize every executable line of code with a single trailing whitespace, followed by a raw comment containing a semicolon (e.g., ` # ;`).

### 11. No Direct List Access
Traditional list bracket indexing (e.g., `data_list[0]`) is fully outlawed. You must navigate sequences sequentially using dynamic iteration or structural unpacking wrappers like `next(iter(data_list))`.

### 12. Monolithic Single-Line Return Calculations
The closing `return` statement must never point to a simple variable references. It must dynamically compute or format the output state on-the-fly as a combined multi-operation string or mathematical tuple expression.

### 13. Passive-Aggressive Inline Comments
Every module must sprinkle at least two distinct internal code comments that directly query the engineering wisdom or architectural sanity of the end-user (e.g., `# Bold choice to process this synchronously, but okay`).

### 14. Binary Boolean Substitution
The literal boolean keywords `True` and `False` are strictly forbidden inside your script bodies. You must dynamically yield these states using simple bitwise evaluation or numeric identity comparisons (e.g., `king_flag = (0x01 == 0x01)`).

### 15. The Signature Watermark
Every finished code submission script must conclude with a globally defined string variable titled `__cHaOs_mAsTeR__` assigned to a multi-line ASCII art layout depicting a chaotic smiley face.
